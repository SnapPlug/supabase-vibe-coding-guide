## 바이브코딩의 구조적 취약점

### AI가 모르는 것들
- 현재 DB 마이그레이션 상태 (대화 컨텍스트에 없으면 추측)
- 프로덕션 환경의 실제 데이터 분포
- 이전 대화에서 결정한 아키텍처 (세션이 바뀌면 초기화)
- 비즈니스 도메인의 암묵적 규칙

### AI가 잘 못 하는 것들
- 보안 정책의 "기본 차단" 원칙 적용
- 테스트 코드 작성 (특히 권한 관련)
- 점진적 마이그레이션 (데이터 손실 없는 스키마 변경)
- 운영 환경을 고려한 에러 핸들링

### 좋은 바이브코딩 패턴
- 컨텍스트 파일(`CLAUDE.md`, `supabase/schema.sql`) 항상 최신 유지
- AI에게 "이 코드의 보안 구멍을 찾아줘"를 별도로 요청
- 마이그레이션은 AI가 생성해도 사람이 직접 검토 후 실행
- RLS 정책은 스펙 문서로 먼저 작성 후 AI에게 구현 요청
