# 06. 스토리지 (Storage)

파일 업로드와 접근 제어. Storage 버킷의 public/private 설정과 RLS 정책이
뒤엉켜 의도치 않게 파일이 공개되거나 접근 불가 상태가 된다.

## 문제 목록

| # | 제목 | 심각도 | 상태 |
|---|------|--------|------|
| - | (문제 발견 시 추가) | - | - |

## 자주 발생하는 패턴

- 버킷을 public으로 만들어놓고 민감한 파일 저장
- Storage RLS와 DB RLS를 별개로 관리하지 않음
- 파일 경로에 사용자 ID를 포함하지 않아 타인 파일 접근 가능
- 대용량 파일 업로드 시 Supabase 기본 제한(50MB) 초과 처리 누락
