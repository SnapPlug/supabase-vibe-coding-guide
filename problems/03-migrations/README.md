# 03. 마이그레이션 (Migrations)

스키마 변경을 추적하고 안전하게 적용하는 것. 바이브코딩에서는 AI가 직접 스키마를 수정하거나
마이그레이션 파일 없이 변경을 적용하는 경우가 많아 이력 관리가 엉망이 된다.

## 자주 발생하는 패턴

- 마이그레이션 파일 없이 Supabase 대시보드에서 직접 스키마 수정
- `supabase db push` vs `supabase migration new` 혼용
- 로컬/스테이징/프로덕션 스키마 불일치
- AI가 현재 마이그레이션 상태를 모르고 이미 존재하는 컬럼을 추가하려는 문제
