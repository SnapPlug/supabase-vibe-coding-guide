# 07. 실시간 (Realtime)

DB 변경사항을 실시간으로 구독하는 기능. 구독 누수, RLS와의 상호작용,
성능 문제 등이 자주 발생한다.

## 문제 목록

| # | 제목 | 심각도 | 상태 |
|---|------|--------|------|
| - | (문제 발견 시 추가) | - | - |

## 자주 발생하는 패턴

- 컴포넌트 언마운트 시 구독 해제(`unsubscribe`) 누락 → 메모리 누수
- RLS 정책이 Realtime 이벤트에도 적용됨을 모르고 설계
- `postgres_changes` vs `broadcast` 잘못 선택
- 동일 채널에 중복 구독
